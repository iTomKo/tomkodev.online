# Config Walkthrough

There are 10 folders and 2 additional files to be set-up. Tottaling in 26 files.

That might sound scary, but most are modified just once in your server existence :D


TIP: In need of the original configs? Go to this guide!


[!] Its programmatically managed. [*] Its just a template for other configs.
[1] Its planned to be converted into centralized database. [2] Its locale file, meant to be modified to match your needs

Folders and their files, including their purpose (click for more info):

./

  • config.yml - General OTTO settings
  • commands.yml - Settings for each avalaible command

./core

Contains Core files.

  • motd.yml - Adding/Removing server motds
  • permissions.yml - Adjusting required permissions for some action
  • servers.yml - Creating server travel aliases. e.g.: /survival
  • whitelist.yml - Adding/Removing servers to be whitelisted via global whitelist toggle

./discord

Stores files required for 100% functionality of Discord Bot

  • cache.yml - [!] Do not edit; Linked User cache
  • discord.yml - Contains required information for the Discord Bot
  • roles.yml - Adding new auto-sync discord roles depending on ingame permission

./features

Stores all features (aka modules) and their settings

  • alerts.yml - Here you set durations of all non-preset alerts
  • features.yml - Here you toggle what features you want to be enabled/disabled

    • ./contests

    Stores required data for contests

    • config.yml - [!] Programmatically managed; Here are all contests stored

    ./limitation

    Limitation setting

    • config.yml - Here you specify what commands you want to limit
    • entries.yml - [!] [1] Programmatically managed; Stores data about current use status for each player

    ./logs

    Logging setting

    • main.yml - Here you specify what commands you want to log

    ./statistics

    Stores required data for statistics computation

    • [!] [1] All the files are programmatically managed; Do not modify them, unless told to do so.

./logs

This folder stores logs (messages sent, groupchat actions and commands executed)

./presets

Used for creating simple presets, so you don't have to type out .. manually everytime

  • example_title.yml - [*]; Preset for a simple title.

./translations

Used for translations and changing message pattern for each OTTO action.

  • locale_CZ.yml - [2]
  • locale_EN.yml - [2]

# Locale Selection

Changing locale is easier, thanks to the previous config knowledge.


But first, what locales are currently avalaible (original)?

  • locale_EN
  • locale_CZ

Changing the locale

  1. Navigate to Otto folder.

  2. Open the config.yml with IDE of your choice.

  3. At the top of the file, there will be a "locale" option.

  4. Change its value to any locale you want. Otto looks for the file name, so you can create your own locales!

  5. Reload Otto or Restart the server to apply changes!

# Custom Color Prefabs

What even are color prefabs? Color prefabs can be used ANYWHERE inside Otto.

For example; need to rebrant your server to different color scheme? You don't have to change all the color codes manually! Just change prefabs value and it will change!


They can be also used inside commands!

Example: /staffchat <<primary>>Wow! This message is colored!

NOTE: The tag is replaced with the HEX code, therefore in gradients you don't need to include additional brackets, but in places, that require <,>, you have to.


Where to look?

  • Look inside config.yml!

    • How to add custom colors?

  • Just add another line, that will be have the name of the color and assign its color!

    • Can I change default (MiniMessage) colors?

  • Yes, you can! Do the same you would with custom color, but name it as the MiniMessage color!

    # Commands & Permissions

    Otto consist of lots of different commands and permissions. Some names can be confusing, because they are shortened.

    Due to that, we will now take look at Otto's commands and than permissions.


    Different types of command setting:

    1. Managing their properties - aliases, standalone status
    2. Toggling them via features.yml
    Go back to the main page

    ## Managing Command properties

    Lets open commands.yml located in root directory of Otto

    You probably know what it does just from reading it. But in case you don't, we will go through all the options.

    1. name - defines the name under which the command is registered into md-5's command tables. [!] Experimental
    2. aliases - here you specify aliases for the given command; leave empty - for no alias
    3. standalone - whether the command should be standalone; Standalone commands do not require the /otto prefix
      and can be executed directly via /<command-name>.
    Go back to the main page

    ## Toggling commands via features.yml

    What even are features in the first place?

  • Features, also known as Modules, are package of function mostly combined with command.

    • Toggling some features on/off.

    Open the features.yml located in ./features/ in IDE of your choice.

    Now you can see all the features, some of them being disabled and most enabled.


    : Lets try to enable /helpop!

    Look for "helpop" via CTRL + F.

    Now change the false value into true.

    After reloading Otto or restarting the server, you will now have helpop enabled!


    NOTE: some features require other features to be enabled.

    You can see what features are required via /otto features and hovering over the feature.

    Go back to the main page

    ## Setting up linked roles

    Having discord bot set-up is a prerequisition!


    In the ./discord directory open roles.yml

    Firstly fill in the default section.

    Guide on getting the required Discord IDs (CLICK)

    From the config you get the most info about the linkage, but here it will be more descriptive.

    1. <name of the role (only for impoved readiability)>

    2. id: <id of the role from discord>

    3. permission: <required permission, for player to be given this role>

    TIP: Making the name of the linked role more descriptive makes it easier later on to debug or change something.

    TIP: Want to apply changes to the discord role linkage without server restart? Reload discord features via /otto dreload.

    NOTE: The role WILL be removed after next player's login to the server without the required permission.

    NOTE: The role WILL be added after first login with the required permission.

    ## What is Admin Panel?

    Having discord bot set-up with admin-panel feature enabled is a prerequisition!


    Admin Panel, as name suggests, is panel meant for server admins.

    When enabled, just simply type .panel into any channel inside your Discord Server!

    [!] NOTE: Anyone can "touch" the panel, therefore DO NOT create it inside public channels!
    Only users with server administration permission can create the panel

    ## Usefull commands

    These are usefull commands and features connected with discord!

    Required feature discord-bot to be enabled, inluding successfull discord bot boot


    /otto info <nick>

    • /otto dreload <bot>

  • Lets you reload discord related configs or/and discord bot (determined by having the bot tag.

  • Lets you see same information as ingame /otto info
